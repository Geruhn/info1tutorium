Widening conversion

byte -> short -> int -> long -> float -> double
>>in dieser Richtung ist implizites Casten möglich>>

Der Grund hierfür ist einfach:

  • Speicher wird immer größer
  • keine Genauigkeit geht verloren

Narrowing conversion

byte <- short <- int <- long <- float <- double
<<in dieser Richtung ist nur explizites Casten möglich<<

Der Grund hierfür ist einfach:

  • Speicher wird immer kleiner
  • Genauigkeit kann verloren gehen

  • Breche überlange Ausdrücke vor dem Operator mit der schwächsten Bindung um
  • Vergleiche Boolesche Werte nicht mit ==
    statt esRegnet == true schreibe esRegnet
    und statt esRegnet == false schreibe !esRegnet
  • Wähle für einen Ausdruck die selbe Operatorenreihenfolge wie in einem mathematischen Ausdruck, implementiere 0 < i < j < n als 0 < i && i < j && j < n
  • Vermeide Seiteneffekte
    int a;
    a = (a = 2) + (a += a) * (a = 1 + a);
  • Breche überlange Ausdrücke vor dem Operator mit der schwächsten Bindung um

if

if (Boolscher Ausdruck) {
	doSomething();
}

doSomething() wird ausgeführt, wenn der boolsche Ausdruck true ist

Ein Beispiel: 


int number = 5;
if (number % 2 == 0) {
	System.out.println("Diese Zahl" 
		+ "ist [...]");
}

if-else


int number = 5;
if (number % 2 == 0) {
	System.out.println("Diese Zahl "
		+ "ist gerade.");
} else {
	System.out.println("Diese Zahl" 
		+ "ist ungerade.")
}

Ein Beispiel: 


char confirmation = 'n';
if (confirmation == 'y' 
	|| confirmation == 'Y') {
	makeSandwich();
} else {
	makeSandwich(sudo);

Boolscher Ausdruck

Aufgabe

Schreibe einen boolschen Ausdruck(battlefield4IsReleased), welcher aussagt, ob Battlefield schon released ist:

  • Der 31. Oktober ist vorbei (int today = 20131024) und
  • Entwickler wurden fertig (boolean developersAreDone = false)
oder
  • Publisher ist geldgierig (boolean publisherIsGreedy = true) und
  • Kontostand der Entwickler sank unter 5 Millionen (double bankaccountDICE = 3943115.79)

Lösung


int today = 20131024;
boolean developersAreDone = false;
boolean publisherIsGreedy = true;
double bankaccountDICE = 3943115.79;

boolean battlefield4IsReleased =
	((today > 20131031 
		&& developersAreDone)
	|| (publisherIsGreedy 
		&& bankaccountDICE < 5000000L));
